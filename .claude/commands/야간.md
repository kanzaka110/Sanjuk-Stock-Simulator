> ⚠️ **시각 표기 필수 규칙**: 서버는 UTC — 브리핑 제목/본문의 시각은 반드시 KST로 변환할 것.
> Python: `from config.settings import KST; from datetime import datetime; datetime.now(KST).strftime("%Y-%m-%d %H:%M")`
> Bash 확인: `TZ=Asia/Seoul date`. naive `datetime.now()` 사용 금지 (9시간 오차).


# 야간 프리브리핑 (내일 한국장 대비)

/한국장 브리핑과 동일한 9단계 분석을 수행하되, **내일 장 시작 전 지정가 예약 주문**에 초점을 맞춥니다.

**핵심 차이점:**
- 모든 매수 추천에 **구체적 지정가** 필수 (범위가 아닌 조건부 가격)
- **갭 시나리오별 대응** (갭업/보합/갭다운)
- **"내일 예약 주문 요약"** 테이블 포함 — MTS/HTS 바로 입력 가능
- 오늘 미국장 마감 결과가 내일 한국장에 미칠 영향 분석

---

## 분석 수행

/한국장 브리핑의 0~9단계를 동일하게 수행합니다. (뉴스 → 레짐 → 시장환경 → P&L → 기술지표 → 리스크 → 백테스트 → KRX → 펀더멘털 → 메모리)

**모든 스크립트는 /한국장 스킬과 동일하게 실행하세요.**

## 출력 특화 규칙 (야간 프리브리핑)

### 결론 포맷

브리핑 최상단에 **내일 예약 주문 요약 테이블**을 배치:

```
🌙 내일 예약 주문 요약 (MTS/HTS 입력용)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

종목명          | 계좌   | 지정가        | 수량  | 유효시간      | 조건
삼성전자        | [ISA]  | ₩58,000 이하  | 10주  | 09:00~15:30  | 갭다운 시 ₩56,000
한화에어로      | [ISA]  | ₩1,280,000   | 1주   | 시초가       | —

📊 갭 시나리오
• 갭업 +1% 이상: 추격 매수 자제, 눌림목 대기 (₩XX,XXX)
• 보합: 위 지정가 유지
• 갭다운 -1% 이상: 하향 지정가 (₩XX,XXX), 추가 매수 기회
```

### 분석 필수 규칙 (기본 규칙 + 야간 추가)

- 모든 매수 추천에 **"₩XX,XXX 이하" 형식 지정가** 필수
- 주문 유효 시간 명시 (시초가 / 전일 / 09:00~09:30 등)
- 가격이 이미 올라 매수 타이밍을 놓친 경우 → **다음 눌림목 가격대** 제시
- 계좌별 규칙, B안 전략, RIA 규칙은 /한국장과 동일 적용

### Notion + 텔레그램 전송

/한국장과 동일한 10단계(Notion 저장 → 텔레그램 전송) 수행. 텔레그램 메시지 헤더에 🌙 표시.
